Deflate 압축 알고리즘에서 악성코드 주입 취약점 분석

Journal of The Korea Institute of Information Security & Cryptology, VOL.32, NO.5, Oct. 2022
김정훈 (Jung-hoon Kim) - BinaryLab CEO / Researcher
요약: 본 연구는 매우 대중적인 Deflate 알고리즘의 비 압축 블록(NPNCB)과 바이트 정렬을 위한 DBA(Disposed Bit Area) 영역을 활용하여 악의적인 데이터나 코드를 은닉하는 취약점을 분석합니다. 실험을 통해 상용 압축 프로그램의 탐지 기법을 우회하고 임의의 코드를 실행할 수 있음을 입증하였습니다.

1. Deflate 알고리즘의 구조적 취약점

Deflate 알고리즘(RFC 1951)은 데이터의 특성에 따라 세 가지 블록 유형을 생성합니다: Dynamic Huffman, Fixed Huffman, 그리고 Non-compressed(비 압축) 블록입니다[cite: 1, 4].

DBA (Disposed Bit Area) 개념

비 압축 블록은 바이트 정렬(Byte Align)을 위해 헤더 내부에 무시되는 비트 영역을 포함합니다. 이를 본 연구에서 DBA라고 명명하였습니다[cite: 10, 11].

Fig. 1. 비 압축 블록의 일반 구조와 DBA 영역

RFC 1951 표준은 이 영역에 무엇을 채워야 하는지 명시하지 않으며, 일반 디코더는 이를 무시(ignored)하고 넘어갑니다[cite: 38].

2. NPNCB (No-Payload Non-Compressed Block)

NPNCB는 원본 데이터(Literal Data)를 포함하지 않는 비 압축 블록으로, LEN(데이터 길이) 값이 0인 특수한 형태입니다[cite: 10, 101].

Fig. 2. 데이터가 없는 NPNCB의 구조
  • 은닉 원리: NPNCB를 정상 블록 사이에 삽입하여 DBA 영역에 악성 비트 주입[cite: 12].
  • 탐지 우회: 압축 해제 후 생성되는 데이터가 없으므로 CRC(Cyclic Redundancy Check) 값에 영향을 주지 않아 상용 백신 및 압축 프로그램을 우회합니다[cite: 143, 266].

3. 공격 사례 및 실험 결과

본 연구에서는 7,680바이트의 악성 윈도우 응용 프로그램을 분해하여 15,360개의 NPNCB 내 DBA에 나누어 주입하였습니다[cite: 299, 328].

ABSTRACT: This research analyzes a vulnerability that hides malicious code using No-Payload Non-Compressed Blocks (NPNCB) and Disposed Bit Areas (DBA) in the popular Deflate algorithm. Experiments demonstrate that contaminated files can bypass commercial security programs and execute arbitrary code via a malicious decoder.

1. Structural Vulnerability of Deflate

The Deflate algorithm (RFC 1951) categorizes compressed data into three types: Dynamic Huffman, Fixed Huffman, and Non-compressed blocks[cite: 27, 31].

The Concept of DBA (Disposed Bit Area)

Non-compressed blocks include a bit area utilized solely for byte alignment, which this study defines as the DBA[cite: 11, 15].

Fig. 1. Structure of Non-compressed block and DBA

Standard decoders ignore the data in the DBA per RFC 1951, providing a blind spot for security inspection[cite: 38].

2. Mechanism of NPNCB Injection

An NPNCB is a block variant with zero literal data length (LEN = 0)[cite: 14, 101].

  • Steganography: Malicious code is fragmented and embedded into the DBAs of multiple NPNCBs inserted between legitimate blocks[cite: 14, 18].
  • CRC Bypass: Since NPNCBs yield no output upon decompression, they do not alter the CRC (Cyclic Redundancy Check) value, evading modification detection[cite: 17, 143].

3. Experimental Proof of Concept

A 7,680-byte test malware was successfully hidden within 15,360 NPNCBs[cite: 299, 328]. Major commercial tools like Bandizip and ALZip failed to detect the injection, while a dedicated malicious decoder reassembled and executed the payload flawlessly[cite: 266, 336, 388].
