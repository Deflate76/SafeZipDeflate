NPNCB Vulnerability Analysis

Malicious Code Injection in Deflate Compression Algorithm

Research Overview

This research by Jung-hoon Kim (BinaryLab CEO) identifies a critical vulnerability in the Deflate algorithm (used in ZIP, APK, PDF). It focuses on the NPNCB (No-Payload Non-Compressed Block), which can be used to hide malicious code without affecting the file's integrity[cite: 4, 10].

The Core Mechanism: DBA

The vulnerability resides in the DBA (Disposed Bit Area) within the header of a non-compressed block. This area exists solely for byte alignment[cite: 11].

  • Stealth: Data in DBA is ignored by standard decoders[cite: 38].
  • Bypass: Since NPNCB has no literal data, it does not change the CRC value after decompression, making it invisible to security software[cite: 143, 266].

Attack Scenario

An attacker can inject thousands of NPNCBs. While a normal program extracts the original data perfectly, a malicious decoder can extract and recombine bits from the DBA to execute arbitrary code in memory[cite: 150, 290].

연구 개요

김정훈(바이너리랩 대표이사) 저자의 연구에 따르면, ZIP, APK, PDF 등에 널리 쓰이는 Deflate 알고리즘에서 심각한 취약점이 발견되었습니다. 핵심은 원본 데이터가 없는 비 압축 블록(NPNCB)을 활용하여 정상 데이터 사이에 악성 코드를 은닉하는 기술입니다[cite: 4, 10].

핵심 원리: DBA (Disposed Bit Area)

비 압축 블록의 헤더에는 바이트 정렬(Byte Align)을 위해 존재하는 DBA 영역이 있습니다[cite: 11].

  • 은닉성: 일반적인 디코더는 DBA 영역의 데이터를 무시하고 건너뜁니다[cite: 38].
  • 우회성: NPNCB는 압축 해제 시 생성되는 데이터가 없으므로, 해제된 데이터 기반의 CRC 체크를 완벽하게 우회합니다[cite: 143, 266].

공격 시나리오

공격자는 수만 개의 NPNCB를 주입할 수 있습니다. 상용 프로그램은 경고 없이 정상 파일을 보여주지만, 악의적 디코더는 이 DBA 조각들을 재조합하여 메모리 상에서 악성 코드를 실행합니다[cite: 150, 290].
